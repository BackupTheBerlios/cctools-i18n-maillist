I think best to ship with all languages included, yes setting up nightly
builds would be awesome (could use sf.net's compiler farms on diff.
architectures), and setting up ccPublisher with standard Open Source
packaging would be desirable to ease this release process.

ccHost is down to a simple:

    make dist

Which relies on a generic makefile I built for the project to deal with
packaging (which could easily be re-used) and puts RPM, tarballs,
bzip2balls, and zip packages into a packages/ folder.

The main deterrent from too often releases is getting the press release
and readme written up (and oh, the testing)...
